_Generated by reading the live repo at `/Users/apple/Downloads/pocketcode` on 2026-07-09._

## What it is
Control VS Code/Cursor from your phone, peer-to-peer, no relay server owned by you.
Two halves:
- **`extension/`** — Node/TS VS Code extension. Binds a loopback HTTP+WS server, exposes PTY/files/git/dev-servers/snapshots over one WebSocket.
- **`android/`** — Kotlin/Jetpack Compose app. Pairs via QR, talks the same WS protocol.

No REST surface beyond `GET /api/health`. **Everything is one WebSocket** at `wss://host:port/ws?token=...`, message shape defined by the discriminated union `WsMsg` in `extension/src/server/protocol.ts` (this file is meant to be the shared source of truth — copy it into Android if a message type changes).

## Wire protocol cheat sheet
| Prefix | Dir | Purpose |
|---|---|---|
| `term.*` | both | PTY open/input/resize/close/data/exit/list |
| `fs.*` | client→server | tree/read/write/mkdir/rename/delete/search |
| `git.*` | client→server | status/diff/stage/unstage/commit/push/pull/branches/checkout/log |
| `devservers`, `devserver.start/stop/log` | client→server | lsof-discovered ("unmanaged") + spawned ("managed") dev processes |
| `workspace.*` | client→server | list/switch VS Code folders |
| `snapshot.*` | client→server | pre-agent `git stash` snapshots |
| `agent.approve` / `agent.reject` | client→server | writes `"y\n"`/`"n\n"` to the most-recently-active alive PTY tab — this is how phone taps answer a CLI agent's y/n prompt |
| `agent.event` | server→client | pushed activity notification |
| `token.refresh` | server→client | proactive renewal, no re-scan needed |

## Auth / pairing (in-memory only, by design)
- `Auth` class (`server/auth.ts`) keeps `sha256(token) → {exp, devices}` in a `Map`. Nothing touches disk server-side.
- First WS connect with a valid token **binds** a device fingerprint. Same token + different fingerprint later → rejected (`device.fingerprintMismatch`), no override flow.
- `renewIfExpiring`: if <6h left, mints a new token, migrates device bindings, keeps the old token valid 2 more minutes so in-flight reconnects don't drop.
- Server binds `127.0.0.1` only (`rebindAll()` flips to `0.0.0.0`, but only used once for the tailscale-ip direct mode).
- Android side stores the token in `EncryptedSharedPreferences`. Session/agent-event history in Room **is stored in plaintext** — flagged in `docs/SECURITY.md` as an unimplemented hardening item, not a bug to "fix silently" without knowing that's intentional-for-now.

## PTY approach — read this before touching `pty/manager.ts`
They deliberately **do not use `node-pty`** (native module breaks across every Cursor/VS Code Electron ABI bump). Instead: `pty-helper.py` calls `openpty()` and forks the shell onto the slave side, relayed via piped stdio. Also deliberately not `script -q /dev/null` — that calls `tcgetattr(stdin)` which fails because the extension host's stdin is a pipe, not a tty.
- `PtyManager` looks for the helper at `out/pty/pty-helper.py` first, falls back to `src/pty/pty-helper.py` in dev.
- Resize mid-session is done by writing the `\x1b[8;{rows};{cols}t` escape into the shell's stdin (not a real TIOCSWINSZ from the Node side) — a shell-honored convention, not a syscall.
- Windows has **no PTY shim** — falls straight to a raw (non-PTY) shell spawn. Known gap, not yet solved.

## Tunnel providers
Preference order: `devtunnel` (default, Microsoft-hosted, persistent named tunnel so reconnects don't need a fresh QR) → `tailscale` (serve mode, your own tailnet) → `tailscale-ip` (direct, flips bind to `0.0.0.0`) → `ssh` (`ssh -R 0:localhost:<port> <target>`, power-user/air-gap option). The extension itself **never opens a public socket** — always forwards a loopback port through one of these.

## Build order (from README)
1. `cd extension && npm install && npm run build && npx vsce package && code --install-extension ...`
2. Set up a tunnel (Tailscale recommended, or devtunnel, or SSH)
3. `cd android && ./gradlew :app:assembleDebug && adb install ...`
4. VS Code → **RemoteDev: Start Mobile Session** → scan QR from PocketCode Android app
